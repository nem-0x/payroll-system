# 学生アルバイト給与計算システム　システム構成図

## 1. システム概要

本システムは、Google Workspace環境上に構築する給与計算自動化システムである。Google Apps Script（GAS）を処理基盤とし、Googleフォーム（入力）、Googleスプレッドシート（データストア・UI）、Slack API（通知）、入退くんAPI（勤怠データ取得）、freee人事労務API（外部連携）を統合する。

システムは5つの層で構成される。

| **層** | **技術要素** | **役割** |
|---|---|---|
| 入力層 | Google Forms / 入退くんAPI | 日報データ・勤怠打刻データ・シフト表・勤怠管理データの取得 |
| 処理層 | Google Apps Script | バリデーション、給与計算、未提出検知、freee連携の全処理を実行 |
| データ層 | Google Spreadsheet | マスタデータ管理、集計結果格納、人間による確認・修正のUI |
| 通知層 | Slack API | 不備通知、未提出リマインド、深夜割増アラートの送信 |
| 外部連携層 | freee人事労務API | 確定済み勤怠データのPUT、給与明細発行（freee既存機能） |

## 2. 構成図の説明

入力層→処理層→データ層→通知層/外部連携層のデータの流れを示す。処理層はPhase 1（不備検知・リマインド）、Phase 2（集計自動化）、Phase 3（freee連携）の3フェーズに分かれ、段階的に機能を追加していく。

入力層の4つのデータソース（オンラインフォーム、オフラインフォーム、入退くんAPI、シフト表/勤怠管理データ）から処理層にデータが流れ、GASが計算・検知処理を実行する。処理結果はデータ層（スプレッドシート群）に格納され、必要に応じて通知層（Slack）や外部連携層（freee）に送信される。

## 5. GASトリガー設計

GAS処理の実行トリガーを以下に定義する。Phase 1のバリデーションとアラートはフォーム送信をトリガーとするリアルタイム処理、それ以外は月次のバッチ処理となる。

| **GAS処理** | **トリガー種別** | **実行タイミング** | **備考** |
|---|---|---|---|
| バリデーション (C-06) | onFormSubmit | フォーム送信時 | オンライン/オフライン各フォームに設定 |
| 深夜割増アラート (C-08) | onFormSubmit | フォーム送信時 | 終了時刻22:10以降を検出 |
| 未提出検知 (C-07) | 手動 or 時間主導型 | 月次（締め日後） | 担当者が実行 or 自動スケジュール |
| 給与計算 (C-09) | 手動 | 月次（締め日後） | 未提出解消後に担当者が実行 |
| freee送信 (C-10) | 手動 | 月次（確定後） | 計算結果確認・確定後に実行 |

> ※ 給与計算（C-09）とfreee送信（C-10）は手動実行とし、担当者の確認ステップを挟む設計とする。
>
> ※ 未提出検知（C-07）は月次で手動実行を基本とし、自動スケジュールへの切り替えも可能。

## 6. GAS実行制約と対策

| **制約** | **上限** | **対策** |
|---|---|---|
| スクリプト実行時間 | 6分/実行 | 100名弱の処理を6分以内に。超過時はバッチ分割 |
| トリガー実行間隔 | 最短1分（時間主導型） | 未提出検知は月次なので問題なし |
| URLFetch呼び出し | 20,000回/日 | API呼び出し回数を監視。Slack/freee/入退くん合計 |
| スプレッドシートセル数 | 1,000万セル/ファイル | 月次データを年度ごとにファイル分割する運用で対応 |

> ※ 100名弱×6業務の月次計算は、テスト段階で実行時間を計測し、6分以内に収まることを確認する。

## 7. セキュリティ・権限設計

1. スプレッドシートのアクセス権限は最小権限の原則に基づき設定する。マスタデータ・集計結果は情シス担当のみ編集可、教室長は閲覧のみとする。
2. freee APIのOAuth2トークンはGASのプロパティストアに格納し、スクリプト外部からアクセスできない形で管理する。
3. Slack通知は個人DMで送信し、給与情報がチャンネルに流れないようにする。
4. 入退くんAPIの認証情報も同様にGASのプロパティストアで管理する。

## 8. 運用フロー（月次サイクル）

Phase 2完了後の月次運用サイクルを以下に示す。

1. **月中:** アルバイトが日報を随時提出。GASが即時バリデーション＋深夜アラートを実行。
2. **月末締め:** 担当者がGASの未提出検知を実行。未提出者にSlackリマインド送信。
3. **未提出解消後:** 担当者がGASの給与計算を実行。集計結果SSに結果出力。
4. **確認・修正:** 担当者がスプレッドシート上で結果を確認。修正があれば理由を記録して修正。確定ステータスを「確定」に変更。
5. **freee入力:** Phase 2では手入力、Phase 3ではGASのfreee送信を実行。
6. **給与明細発行:** freee上で明細を確定・発行。
