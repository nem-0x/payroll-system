# payroll-system
# 学生アルバイト給与計算システム

Google Workspace環境上に構築した給与計算自動化システムの要件定義・設計ドキュメントです。

## 背景・課題

教育系スタートアップにて、情報システム担当として学生アルバイト（約100名）の給与計算業務を一人で運用していました。

月次の給与計算に以下の問題がありました。

- **集計に3日以上かかる**: 6種類の業務区分ごとに異なる計算ロジックを手作業で適用
- **日報の不備対応に工数がかかる**: 未提出・記入ミスの検知と差し戻しがすべて手動
- **freeeへの手入力**: 約100名分のデータを管理画面から1件ずつ入力
- **属人化**: 全工程を1名で担当、バックアップ体制なし

## 解決アプローチ

既存のGoogle Workspace環境を活用し、GASによる段階的な自動化を設計しました。

### 3フェーズ実装計画

| フェーズ | 内容 | 目標効果 |
|---|---|---|
| Phase 1 | 日報バリデーション・未提出検知・Slack通知 | 不備率50%以上削減 |
| Phase 2 | 業務A〜F給与計算自動化・入退くんAPI連携 | 集計作業を半日〜1日に短縮 |
| Phase 3 | freee人事労務API連携による手入力廃止 | freee手入力ゼロ化 |

### 技術スタック

- **Google Apps Script**: メイン処理基盤（バリデーション・計算・API連携）
- **Google Spreadsheet**: データストア・マスタ管理・確認UI
- **Slack API**: 不備通知・未提出リマインド・深夜割増アラート
- **入退くん API**: 勤怠打刻データ取得（Phase 2）
- **freee人事労務 API**: 勤怠データPUT・OAuth2認証（Phase 3）

## 設計上の工夫

**計算ロジックの複雑さへの対応**

時給制・コマ給・回数制が混在する6業務区分に対し、単価マスタをスプレッドシートで管理する設計を採用。給与体系の変更時にコード修正なしで対応できる構造にしました。

**端数処理の統一**

各項目の計算途中では端数を保持し、月次総支給額の算出時に1回だけROUNDUPを適用。業務Dのコマ手当（キャンセルコマにも適用）など業務固有の例外ロジックを仕様書に明文化しました。

**段階的導入によるリスク低減**

既存業務を止めずに導入できるよう、3フェーズに分けて効果を確認しながら進める計画を設計しました。
